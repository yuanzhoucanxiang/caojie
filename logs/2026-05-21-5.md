## 犯的错误与避免方法

### 错误 1：Shader 初始就显示遮罩（3次修复）
**表现**：游戏一启动，画面就在遮罩中。
**原因**：shader 中 `eff = edge_dist - n * edge_width` 可以产生负值。`step(负值, 0)` 返回 1，即使 dissolve=0 也显示遮罩。
**修复**：改用 `coverage = (0.5 - edge_dist) + n * 0.15`，保证 coverage >= 0。`step(coverage, 0)` = 0。
**避免方法**：
- shader 中 `step(a, b)` 要注意 a<=b 返回 1，中间变量可能为负
- 遮罩 shader 在 dissolve=0 时必须输出 alpha=0，设计时从这个边界条件反推公式
- 测试时先看 dissolve=0 的状态是否正确，再看过渡过程

### 错误 2：水彩过渡效果不明显
**表现**：场景切换时看不太到水彩效果。
**原因**：初始参数太小（noise_scale=4, edge_width=0.08），过渡太快（0.5s）。
**修复**：增大参数（noise_scale=6, edge_width=0.15），过渡时间加长到 0.8s。
**避免方法**：特效参数先做大再缩小，比先做小再调大容易调试。

### 错误 3：过渡闪烁两下
**表现**：场景切换时画面闪两下，不平滑。
**原因**：过渡用了 `TRANS_QUAD` + `EASE_IN/EASE_OUT`，速度曲线有突变，而且没有在完全覆盖后停顿就直接切换。
**修复**：
- 改用 `TRANS_SINE` + `EASE_IN_OUT`，速度曲线更圆滑
- 水彩完全覆盖后加 0.15s 停顿，确保场景在遮罩下无声切换
**避免方法**：过渡动画用正弦曲线（TRANS_SINE）最平滑，避免二次/三次曲线的加速突变。

### 错误 4：褪色遮罩用了纯白色
**表现**：用户反馈过渡时颜色不对。
**原因**：`_fade_overlay` 初始颜色设为 `Color(1, 1, 1, 0)`。
**修复**：改用场景主色调 `transition_color`，游戏中不出现纯白纯黑。
**避免方法**：美术风格规则（不出现纯白纯黑）要写入代码规范，每写新代码前先检查。
