### 19. 场景过渡系统 Bug 修复——双滑问题

故障现象：1 楼切换到室外时幕布滑两下反弹，其他方向正常。

排查路径（按时间顺序）：

| 尝试 | 结果 |
|------|------|
| `camera.reset_smoothing()` | 无效 |
| 加 `await create_timer(0.12)` 给 GPU 缓冲 | 无效 |
| 入树前预配置相机 zoom/offset/limits | 更严重 |
| 瞬时开关幕布（去掉 tween 动画调试） | 闪烁仍在，但排除了 tween |
| 新旧场景共存（延迟删旧场景） | 导致无法出门 |
| 关掉 `position_smoothing_enabled` | 无效 |
| 改成单滑过渡（幕布直接遮挡→加载→滑出） | 绕过了问题但未定位根因 |

**最终定位**：幕布 ColorRect 之前挂在 SceneManager（普通 Node）下，跟 2D 场景在同一渲染管线。从室内 Polygon2D 场景切到庭院 ParallaxBackground 场景时，相机 zoom 从 1.8 跳到 1.25，视差层重算，幕布的位置更新和场景渲染在同一个 GPU pass 里互相干扰，导致幕布位置回弹。

期间在修"幕布底部有缝隙"时给幕布加了独立 `CanvasLayer(layer=128)`，但没意识到它顺带治了双滑问题。确认后恢复双滑版本（滑入→切场景→滑出），一切正常。

**教训**：UI 遮罩层必须放独立 CanvasLayer，脱离 2D 渲染管线，不受相机/视差/场景切换的任何影响。这是 Godot UI 系统的基本常识，但排查时忽略了。

**回退到单滑版本的中间提交**保留在 git 中：`1bb711f`，可对比参考。

## 犯的错误

### 双滑问题排查方向全错
- 花了大量时间在 tween 时序、camera smoothing、GPU 帧等待上
- 根因是 CanvasLayer 隔离问题，之前顺手修了缝隙但没意识到顺带治了双滑
- 单滑版本提交到 git 后才在后面对比中确认 CanvasLayer 才是关键
- 教训：排查 UI 相关问题第一反应应该是渲染层隔离

### 新旧场景共存的过渡方案导致功能破坏
- 尝试让旧场景 hidden+disabled 延迟删除来避免 GPU 冲突
- 导致 1 楼出门功能完全失效
- 双场景共存时 `current_scene` 切换逻辑不明确

### 入树前预配置相机反而更糟
- 在场景 `add_child` 前设 zoom/offset，期望首帧就用正确参数
- ParallaxBackground 的初始化流程被打乱，导致更严重的视觉问题
