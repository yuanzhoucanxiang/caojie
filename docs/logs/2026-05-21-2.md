### 3. 画面布局设计

#### 场景分层（5层视差结构）
- 天空层 → 远景层 → 中景层 → 场景层 → 前景层
- 玩家在场景层上，可前后移动（深度缩放）
- 参考：Ori 的多层视差滚动

#### 室外/室内视角切换（用户提出）
- 室外：较远镜头，多层视差，开阔感
- 室内：拉近镜头，聚焦角色，亲密感
- 天台作为"过渡空间"——物理上在室内，视觉上是室外
- 参考：《直到那时》的室内拉近处理

#### 摄像机规则
- 水平跟随玩家，平滑滚动
- 场景有左右边界，触发场景切换
- 室内外切换有平滑过渡动画

### 4. UI 设计重写

#### 无常驻 HUD（用户提出）
- 画面完全干净，不显示任何常驻信息
- 属性变化不弹通知，玩家自行打开菜单查看
- 理念：通过感受理解角色变化，不看数字

#### 漫画框式对话（用户提出）
- 对话不是传统底部对话框，而是漫画气泡从角色旁边冒出
- 选项也是漫画风格小气泡，非传统按钮
- 4种对话框类型：普通气泡、放大框、特写框、内心框
- 参考：漫画/日漫的对话呈现方式

#### 角色表情差分（用户提出，参考《爱氏物语》）
- 对话时展示角色的像素表情差分
- 9种基础表情：普通、微笑、大笑、皱眉、生气、难过、惊讶、害羞、无语
- 全部12个角色的表情需求表
- 表情差分精度高于场景中的像素小人
- 参考：《爱氏物语》(Eastward) 的对话演出

### 5. 场景切换设计

#### 像素化水彩过渡（用户提出水彩想法 + 方案A）
- 水彩晕开感但保持像素网格（dithering处理）
- 用像素化的流动效果模拟水彩，和漫画框风格统一
- 不同目的地用水彩颜色区分（暖黄=出门、青绿=鱼塘、橙红=集市、蓝灰=回房间）
- 参考：《Gris》的水彩过渡效果

## 附：Demo 事件脚本

### 事件1：外婆——择菜
```
【场景：客厅，外婆坐在小竹椅上择菜】
外婆（微笑）："乖孙，来帮外婆择菜。"
  选项A："好的外婆！" → 主角蹲下来帮外婆择菜 →【懂事+1，亲密+1】
  选项B："我想出去玩…" → 外婆（普通）"去吧去吧，别跑太远。" →【好奇+1，亲密-1】
```

### 事件2：舅舅——院子里抽烟
```
【场景：院子里，舅舅靠在墙边抽烟】
舅舅（普通）："来了啊。"
  选项A："舅舅好！" → 舅舅（苦笑）"嗯，去玩吧。" →【懂事+1】
  选项B："（点点头）" → 舅舅看了主角一眼，没再说什么，继续抽烟。（无属性变化）
```

### 事件3：舅妈——厨房做饭
```
【场景：厨房，舅妈在柴火灶前炒菜】
舅妈（微笑）："饿了吧？饭快好了，再等一会儿。"
  选项A："谢谢舅妈！" → 舅妈（大笑）"这孩子嘴真甜。去洗洗手，一会儿叫你吃饭。" →【懂事+1，亲密+1】
  选项B："还好，不饿。" → 舅妈（普通）"那行，饿了跟舅妈说。" →【亲密+0】
```

### 事件4：二表哥——叫主角出去玩
```
【场景：院子门口，二表哥带着土狗】
二表哥（咧嘴笑）："走！带你去抓青蛙！"
  选项A："走！" → 二表哥（大笑）"走走走！慢了就抓不到了！" →【好奇+1，体力+1，亲密+1】
  选项B："去哪里抓？" → 二表哥（贼兮兮）"鱼塘那边，水沟里好多！外公不会发现的！" →【好奇+1，亲密+0】
```

### 事件5：小明——村道第一次见面
```
【场景：村道上，小明从对面跑过来】
小明（咧嘴笑）："哎！你是新来的吧？我叫亚明，大家都叫我小明！"
  选项A："我叫___！" → 小明（大笑）"走！我带你去认识其他小朋友！" →【好奇+1，亲密+1】
  选项B："（点点头）" → 小明（普通）"你不爱说话啊？没事，跟我来就对了！" →【亲密+0】
```
