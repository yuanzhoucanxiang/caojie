## 犯过的错误总结（以后避免）

### 1. get_viewport_rect() 只能在 Viewport 上调用
- `get_viewport_rect()` 只存在于 Window/Viewport 节点
- 普通 Node 要用 `get_viewport().get_visible_rect().size`

### 2. 节点加入场景树前不能调用 create_tween()
- `setup()` 中创建节点后立即调用 `create_tween()` 会报 null 错误
- 动画必须在 `_ready()` 中执行（节点已加入树）
- 同理 `get_tree().create_timer()` 也需要节点在树中

### 3. Control 节点没有大小就收不到鼠标输入
- Control 默认大小为 0x0，点击区域不存在
- 用 PanelContainer 或设置 `size` 和 `custom_minimum_size`

### 4. CanvasLayer 子节点的输入传播不可靠
- CanvasLayer 下的 Control 可能收不到 `_gui_input`
- 解决：键盘用 `_physics_process()` + `Input.is_action_just_pressed()`
- 鼠标在父节点 `_input()` 中做位置碰撞检测

### 5. .tscn 文件中的 Color 必须有4个分量
- `Color(0.95, 0.93, 0.88)` 会解析失败
- 必须写 `Color(0.95, 0.93, 0.88, 1)` 包含 alpha

### 6. Autoload 中的 load() 可能找不到资源路径
- `load("res://scenes/xxx.tscn")` 在 Autoload 中可能失败
- 解决方案：用 `@export var path: String` 在 .tscn 中设置路径，在 `_ready()` 中 load

### 7. Player/NPC 放在 CanvasLayer 下会破坏 ParallaxBackground
- CanvasLayer 独立于世界坐标系
- Camera2D 在 CanvasLayer 下不影响 ParallaxBackground
- 解决：Player 和 NPC 直接放在主场景节点树下

### 8. 信号重复连接
- 编辑时不小心复制粘贴了信号连接代码
- 导致回调执行两次，行为异常
- 连接前检查或用 `is_connected()` 判断

### 9. 按键延续问题
- 同一个键从一个状态延续到下一个状态（如按E继续对话→按E确认选项）
- 解决：状态切换后加短暂输入冷却（0.1-0.15秒）

### 10. 交互范围一开始设太大
- NPC InteractionZone 半径 90px + Player InteractionDetector 半径 80px = 有效范围 170px
- 改为各 25px，玩家需要走到面前才能互动
