# 04 - 事件系统

## 事件分类

| 类型 | 触发方式 | 示例 |
|------|----------|------|
| **剧情事件** | 必然触发，推进主线 | 初到乡下、年例、回城读书 |
| **支线事件** | 满足条件时触发 | 帮邻居修屋顶、捡到受伤小鸟 |
| **随机事件** | 一定概率每日触发 | 台风天、赶集日 |
| **日常事件** | 每天固定时段 | 吃饭、起床 |
| **年例事件** | 年例当天固定触发 | 游神、舞狮、摆宴 |

---

## 事件结构

每个事件是一个数据包：

```gdscript
{
    "id": "event_id",           # 唯一标识
    "name": "事件显示名",
    "type": "story/side/random/daily/nianli",
    "scene": "场景ID",         # 在哪个场景发生
    "conditions": [],           # 触发条件列表
    "time_window": "morning/afternoon/evening",  # 可触发时段
    "season": "summer",        # 可选，限定季节
    "dialogue": [              # 对话内容
        {
            "speaker": "NPC_ID",
            "text": "对话内容",
            "choices": [
                {
                    "text": "选项文字",
                    "effects": {"属性": +1},
                    "next": "下一组对话/事件ID"
                }
            ]
        }
    ],
    "rewards": {               # 事件完成奖励
        "attributes": {"属性": +1},
        "unlock_event": "event_id"
    }
}
```

---

## 触发条件类型

| 条件类型 | 语法 | 说明 |
|----------|------|------|
| 前置事件完成 | `event_completed:event_id` | 需要某事件已发生 |
| 属性门槛 | `attr:属性名>=值` | 需要属性达到某值 |
| 时间条件 | `day_count>=N` | 游戏天数达到某天 |
| 季节条件 | `season:summer` | 限定季节 |
| 关系条件 | `relationship:奶奶>=值` | 与某 NPC 关系度 |
| 已发生次数 | `event_count:event_id>=N` | 某事件发生过 N 次 |

## 事件链示例：年例

```
年例前一周（春）:
├── 奶奶通知 → "今年年例就在下周，你去不去看？"
│   ├── "我要去！" → 好奇+1，触发准备事件链
│   └── "不想去..." → 好奇-1，跳过准备事件
│
年例当天:
├── 起床 → 全村准备（准备事件：帮忙抬旗帜/摆供品）
├── 上午 → 游神巡游（观察事件：描述场景，可选择跟随或围观）
├── 中午 → 摆宴（社交事件：和村里孩子一起吃饭，认识新朋友）
├── 下午 → 飘色巡游（观赏事件）
├── 傍晚 → 烟花/鞭炮（自由探索时间）
└── 结束 → 奶奶问"今天好玩吗？"
```
